How It Works
PBKDF2: Your passphrase is never used directly. PBKDF2 (Password-Based Key Derivation Function 2) with SHA-256 and 310,000 iterations + a random 16-byte salt stretches it into a 256-bit key. This makes brute-force attacks computationally devastating.

Random IV: Each encryption generates a fresh cryptographically random 96-bit Initialization Vector. Encrypting the same text twice always produces different ciphertext.

AES-256-GCM: The gold standard. Galois/Counter Mode provides both confidentiality (no one can read it) and authenticity (no one can tamper with it). The 128-bit authentication tag means any modification is instantly detected on decryption — the operation throws an error.

Output format: salt(32) + iv(16) + ciphertext — all concatenated, then Base64 encoded. Fully self-contained; salt and IV are embedded for decryption.

⚠ Your passphrase never leaves your browser. All crypto runs via the native Web Crypto API — no libraries, no network.
